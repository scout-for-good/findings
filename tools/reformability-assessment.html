Is This System Fixable?

A reformability assessment for algorithmic systems

Some algorithmic systems cause harm through their parameters — biased inputs, skewed thresholds, discriminatory training data. Fix the parameters, fix the harm. These systems are reformable.

Other systems cause harm through their fundamental method. The harm isn't a bug in the implementation — it is the implementation. No amount of debiasing, retraining, or parameter adjustment will fix it, because the thing the system does is the thing that's wrong. These systems are unreformable.

The distinction matters enormously for governance strategy. For reformable systems, the goal is forcing change: constrain inputs, mandate audits, require transparency. For unreformable systems, governance should aim at discontinuation — arguing the system shouldn't exist, not that it should be fairer.

This tool walks you through five dimensions that distinguish reformable from unreformable systems. It won't give you a definitive answer — reformability is a judgment, not a measurement. But it will help you ask the right questions and avoid spending years trying to fix something that can't be fixed.

Reformability Assessment

1

Where Does the Harm Originate?

Some harms come from what gets fed into a system. Others come from what the system fundamentally does. The distinction determines whether the harm can be addressed by changing inputs or requires changing the approach entirely.

If you replaced all the training data with perfectly representative, unbiased data, would the system still cause the kind of harm you're concerned about?
Is the harm in a specific parameter or threshold that could be identified and changed?
Could you describe the harm without referring to the system's method — just by pointing to a bad input or configuration?
2

The Debiasing Test

Amsterdam tried to debias a welfare fraud prediction system with genuine commitment, expert help, and iterative testing. Every fix introduced new bias. Some systems resist debiasing because the bias is structural to the method.

If you removed protected characteristics (and their proxies) from the system, would it still serve its stated purpose?
Has anyone actually tried to debias this system (or a closely similar one)? What happened?
Does the system use population-level statistical patterns to make decisions about individuals?
3

Reflexivity — Does It Create What It Claims to Measure?

Some systems don't just measure a condition — they produce or reinforce it. A predictive policing algorithm that sends more officers to a neighborhood finds more crime there, confirming its prediction. This self-reinforcing loop makes reform structurally difficult.

Does the system's output influence the conditions it uses as input?
If you turned the system off for a year and then measured the same outcomes, would the patterns the system claims to detect still be there?
4

Structural Dependency — Who Needs the Harm?

When a system's harmful outputs are load-bearing for someone's revenue, power, or institutional function, reform faces structural resistance. UnitedHealth's recovery prediction algorithm saved money precisely because it underestimated care needs — the harm was the business model.

If the system were reformed to eliminate the harm, would the organization that operates it lose revenue, efficiency, or institutional power?
Was the system acquired, built, or deployed specifically because of the outputs that cause harm?
Has the operator actively resisted or undermined reform efforts?
5

The Amsterdam Test — Could Good Faith Fix This?

Imagine the ideal scenario: the operator genuinely wants to fix the system, has unlimited resources, and brings in independent experts. Would it work? This isolates the structural question from the political one.

If the operator committed fully to reform — unlimited budget, genuine intent, independent oversight — could the system be made to not cause this harm?
Can you describe a version of this system that serves the same purpose without causing the harm?
Is the system doing something that a human process could do without the same harm — even if more slowly or expensively?

This assessment is a structured way to ask important questions — not a definitive judgment. Reformability exists on a spectrum, and reasonable people can disagree about where a system falls. The value of the exercise is in the questions themselves: if they're hard to answer, that's diagnostic. Use this as a starting point for analysis, not a substitute for it.
