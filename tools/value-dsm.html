Value Dependency Matrix

A Design Structure Matrix for mapping where values freeze in your system

In engineering, a Design Structure Matrix maps dependencies between components. When you change one part, the matrix shows what else must change. Engineers use this to find which decisions are load-bearing — deeply frozen into the system's architecture.

This tool applies the same method to value-bearing decisions in any system. Define your components, mark which ones encode value choices, map the dependencies, then simulate changes to see how they propagate.

Click cells to cycle: empty → weak dependency → strong dependency. Right-click a row header to toggle value-bearing status or remove. In Propagation Mode, click a row header to simulate changing that component.
